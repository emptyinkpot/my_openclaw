# 完整润色流程说明

## 🔄 总览：完整流程架构

```
用户输入文本 → 资源库配置 → 步骤配置 → 执行润色 → 实时AI检测 → 效果评估 → 历史记录
     ↓              ↓            ↓            ↓              ↓              ↓           ↓
   原文         词汇/禁用词    21个步骤     SSE流式输出    打字停止检测    对比展示    统计分析
```

---

## 📋 第一阶段：资源库配置（润色前准备）

### 1. 词汇库（Vocabulary）
```
用途：提供同义词替换来源
配置：可添加分类词汇，如"形容词"、"动词"、"名词"等
示例：
- 优秀 → 出色/杰出/卓越/优异
- 重要 → 关键/主要/核心/重大
```

### 2. 禁用词/反感词库（Banned Words）
```
用途：检测并替换劣质表达
配置：{ content: "禁用词", alternative: "替换词", reason: "原因" }
示例：
- 不但...而且 → 既...又... / 非独...亦...
- 在此基础上 → 据此 / 基于
```

### 3. 句式逻辑禁用库（Sentence Patterns）
```
用途：识别并替换禁用的句式表达
配置：{ content: "禁用句式", replacements: ["替换1", "替换2"], reason: "原因" }
支持：模糊识别变体形式
```

### 4. 文献引用库（Literature Resources）
```
用途：提供引用素材
配置：{ type: "book", title: "书名", author: "作者", priority: 1-5 }
引用比例控制：0-10% → 0-1处，10-30% → 1-2处，30-50% → 3-5处
```

### 5. 梗资源库（Memes）
```
用途：融合网络梗/流行语
配置：{ content: "梗内容", category: "分类" }
模式：自然融合 / 反讽优先
```

## ⚙️ 第二阶段：步骤配置（21个步骤）

### 【前置配置阶段】8个步骤

| 步骤 | 功能 | 触发条件 |
|------|------|----------|
| **detect** | 自动识别 | 固定开启，识别叙事角色/年代/阵营 |
| **properNounCheck** | 专有名词检查 | 检测并替换昭和/唐朝/法兰西等现实专有名词 |
| **narrativePerspective** | 叙事视角 | 三种视角：第一人称沉浸/有限上帝/全知上帝 |
| **classicalApply** | 文言化应用 | 根据classicalRatio(0-100%)调整语言风格 |
| **citationApply** | 引用应用 | 根据文献库和引用比例插入引用 |
| **particleApply** | 虚词限制 | 控制之/乎/者/也等虚词使用数量 |
| **punctuationApply** | 标点规范 | 禁用破折号/冒号/括号，转换日式引号 |
| **quoteProtect** | 引用保护 | 保护引号/书名号内的内容不被修改 |

### 【核心处理阶段】5个步骤

| 步骤 | 功能 | 执行逻辑 |
|------|------|----------|
| **polish** | 词汇润色 | 匹配词汇库，执行同义词替换，保护专有名词 |
| **bannedWords** | 反感词净化 | 检测禁用词，按优先级选择替换词（偏好词汇 > 备选词 > 文言建议） |
| **sentencePatterns** | 句式逻辑净化 | 识别禁用句式，选择合适的文言/现代替换 |
| **memeFuse** | 梗融合 | 判断语境契合度，自然植入梗内容 |
| **styleForge** | 风格熔铸 | 统一比喻风格，植入细节描写 |

### 【后处理阶段】2个步骤

| 步骤 | 功能 |
|------|------|
| **markdownClean** | 格式净化，清理**加粗**、~~删除线~~等Markdown语法 |
| **titleExtract** | 标题提取，识别并分离标题和正文 |

### 【审稿验证阶段】5个步骤

| 步骤 | 功能 |
|------|------|
| **semanticCheck** | 语义检查，确保原意不变、无歧义 |
| **finalReview** | 综合审稿，检查引用完整性/专有名词/禁用词处理/风格统一 |
| **wordUsageCheck** | 用词审查，识别替换产生的用词不当 |
| **smartFix** | 智能微调，修正用词错误 |
| **breathSegment** | 呼吸感分段，按朗读节奏调整段落 |

## 🚀 第三阶段：执行润色（API处理流程）

### 进度分配（SSE流式输出）

```
 0% -  5%  初始化处理环境
 5% -  8%  引用保护（quoteProtect）
 8% - 12%  专有名词检查（properNounCheck）
12% - 16%  智能联想扩展资源
16% - 15%  构建处理指令（Prompt）
15% - 70%  LLM处理（主要时间）
           ├─ 短文本：单次处理
           └─ 长文本：分段处理（每段独立进度）
70% - 75%  解析结果
75% - 78%  恢复引用
78% - 88%  验证（词汇+禁用词）
88% - 95%  后处理
95% - 100% 生成报告
```

### 核心Prompt结构

```
【最高优先级】
任务：只做词汇替换，不重写句子
禁止：改变句子结构、改变大意、删除新增内容
保护：引用内容、专有名词、历史术语

【必须保护的专有名词】
- 科技公司：华为/苹果/谷歌/微软...
- 人名地名：[自动识别结果]

【核心配置】
文言化：X% - [程度描述]
标点：[禁用规则]

【处理步骤】
步骤1 → 步骤2 → 步骤3...

【各步骤指令】
[按启用步骤动态生成]

【输出格式】
直接输出处理后的文本
```

## 🔍 第四阶段：实时AI检测

### 触发机制
```javascript
// 防抖检测：打字停止800ms后自动触发
useEffect(() => {
  const timer = setTimeout(() => {
    detect(text);  // 快速检测，只获取分数
  }, 800);
  return () => clearTimeout(timer);
}, [text]);
```

### 检测特征
```typescript
// AI特征计算
{
  sentenceSimilarity: number;   // 句式重复度
  aiWordRatio: number;          // AI常用词比例
  lengthVariance: number;       // 句长变异系数
  emotionDensity: number;       // 情感词密度
  conjunctionDensity: number;   // 连接词密度
}

// 综合AI分数计算
aiScore = 30  // 基础分
  + sentenceSimilarity * 20
  + aiWordRatio * 30
  - lengthVariance * 20
  - emotionDensity * 10
```

### 分数显示
```
🤖 AI检测: 45分
├─ 0-30分: 高度人工化 ✅
├─ 31-50分: 轻度AI痕迹 ⚠️
├─ 51-70分: 明显AI痕迹 ⚠️
└─ 71-100分: 高度AI化 ❌
```

## 🎮 用户操作流程

### 简单模式（一键润色）
```
1. 粘贴文本
2. 点击"开始润色"
3. 等待处理完成
4. 查看结果和AI分数变化
```

### 高级模式（完整配置）
```
1. 配置资源库
   ├─ 添加词汇（分类管理）
   ├─ 添加禁用词（含替换建议）
   ├─ 添加句式逻辑
   ├─ 添加文献引用
   └─ 添加梗资源

2. 配置润色参数
   ├─ 文言化程度：0-100%
   ├─ 叙事视角：第一人称/有限上帝/全知上帝
   ├─ 引用比例：0-100%
   └─ 虚词限制：开/关 + 具体设置

3. 选择处理步骤
   ├─ 启用/禁用各步骤
   └─ 调整步骤顺序

4. 执行润色
   ├─ 实时查看进度
   └─ 观察AI分数变化

5. 效果评估
   ├─ 查看6维度评分
   ├─ 对比原文/润色后
   └─ 查看优化建议

6. 模型训练（可选）
   ├─ 添加标注数据
   ├─ 调整特征权重
   └─ 应用训练后的模型
```
