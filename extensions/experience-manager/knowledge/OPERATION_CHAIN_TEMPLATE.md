## 一、记录模板

每个操作需要记录以下要素：

```markdown
### 操作名称

| 要素 | 内容 |
|------|------|
| **前置操作** | 执行此操作前必须完成的操作 |
| **URL** | 目标页面的URL |
| **入口UI** | 从哪里点击进入 |
| **页面UI** | 页面结构和主要元素 |
| **作用** | 这个操作做什么 |
| **通往** | 执行后可以去哪里 |
| **验证方式** | 如何确认操作成功 |

**关键代码**：
```javascript
// 操作代码
```

**注意事项**：
- 注意点1
- 注意点2
```

## 三、记录示例

### Coze 登录恢复

| 要素 | 内容 |
|------|------|
| **前置操作** | 无（入口操作） |
| **URL** | `https://www.coze.cn/` |
| **入口UI** | 直接访问 |
| **页面UI** | 主页，可能有登录弹窗 |
| **作用** | 设置登录状态，为后续操作做准备 |
| **通往** | 扣子编程、个人中心等 |
| **验证方式** | 检查 localStorage 数量 > 10 |

**关键代码**：
```javascript
// 设置登录状态
await page.evaluate((items) => {
  for (const [k, v] of Object.entries(items)) {
    try { localStorage.setItem(k, v); } catch (e) {}
  }
}, data.localStorage);
await context.addCookies(data.cookies);
```

**注意事项**：
- 必须先访问主站，再跳转子站点
- localStorage 和 cookies 都需要设置
