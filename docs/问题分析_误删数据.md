# 脚本问题分析报告

## 问题概述
导入《枪与凋零之花》细纲时，误删了已存在的正文内容。

## 根本原因

### 1. 脚本设计缺陷 - 全删全插模式

**错误代码**（import_qing.js 第35行）：
```javascript
// 1. 清空旧章节
await conn.execute('DELETE FROM chapters WHERE work_id = ?', [workId]);
console.log('已清空旧章节\n');
```

**问题**：
- 直接删除work_id=7的所有章节记录
- 不检查现有数据是否有content内容
- 不区分是更新细纲还是更新正文

### 2. 数据表职责不清

| 表 | 应该存储 | 我的错误操作 |
|---|---------|-------------|
| chapters | 章节正文(content) | 删除后只插入空内容 |
| chapter_outlines | 细纲(plot_summary等) | 正确插入 |

**错误**：把两个表的操作混在一起，删除chapters时连正文一起删了。

### 3. 缺乏前置检查

**应该做的检查**：
```javascript
// 检查现有数据
const [existing] = await conn.execute(
  'SELECT chapter_number, LENGTH(content) as len FROM chapters WHERE work_id = ?',
  [workId]
);
const hasContent = existing.filter(r => r.len > 0);
if (hasContent.length > 0) {
  console.log(`警告: 已有${hasContent.length}章正文内容，是否覆盖？`);
  // 询问用户或跳过
}
```

**实际做的**：直接DELETE，无任何检查。

### 4. 耦合性过高

**一个脚本做了太多事**：
1. 清空chapters表 ❌
2. 清空chapter_outlines表 ❌
3. 插入新章节（content=''）❌
4. 插入细纲 ✅
5. 插入卷纲 ✅

**应该拆分**：
- 导入细纲脚本：只操作chapter_outlines表，不动chapters
- 导入正文脚本：只更新chapters表的content字段

## 正确的设计

### 方案A：增量更新（推荐）
```javascript
// 只更新/插入，不删除
for (const ch of chapters) {
  // UPSERT语法：存在则更新，不存在则插入
  await conn.execute(
    `INSERT INTO chapters (work_id, chapter_number, title, ...) 
     VALUES (?, ?, ?, ...)
     ON DUPLICATE KEY UPDATE 
     title = VALUES(title), ...`,
    [workId, ch.number, ch.title, ...]
  );
}
```

### 方案B：分离操作
```javascript
// 脚本1：导入/更新细纲（只操作chapter_outlines）
// 不碰chapters表的正文内容

// 脚本2：导入正文（只操作chapters.content）
// 只UPDATE content字段，不动其他数据
```

### 方案C：字段级更新
```javascript
// 如果只更新细纲，完全不动chapters表
await conn.execute(
  'UPDATE chapter_outlines SET plot_summary = ? WHERE work_id = ? AND chapter_number = ?',
  [ch.plot_summary, workId, ch.number]
);
```

## 修复措施

1. **立即**：恢复被删的正文数据（已从txt重新导入）
2. **短期**：重写导入脚本，使用UPSERT代替DELETE+INSERT
3. **长期**：建立数据库操作规范，强制前置检查

## 教训

1. **DELETE是危险的** - 除非确定要清空，否则用UPDATE/UPSERT
2. **先检查后操作** - 任何修改前先查询现有数据
3. **单脚单一职责** - 一个脚本只做一件事，不要混合操作
4. **数据备份** - 大规模更新前先备份
