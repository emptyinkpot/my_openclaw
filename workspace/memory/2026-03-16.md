## 下午：Hook 配置修复

### 发现的问题

1. **HOOK.md 格式错误**
   - 问题：使用 YAML 格式导致 OpenClaw 无法识别事件配置
   - 解决：改为 JSON 格式的 metadata
   ```yaml
   # 错误格式
   metadata:
     openclaw:
       events: ["tool:complete"]
   
   # 正确格式
   metadata:
     {
       "openclaw":
         {
           "events": ["tool:complete"]
         }
     }
   ```

2. **Hook 目录结构问题**
   - 问题：OpenClaw 要求 Hook 必须在 package 目录内
   - 解决：创建 `/workspace/projects/hooks-pack/` 统一存放

3. **lib 依赖缺失**
   - 问题：hooks-pack 中的 Hook 无法找到 `../../lib` 目录
   - 解决：复制 `workspace/lib` 到 `hooks-pack/lib`

4. **extraDirs 配置清理后丢失**
   - 问题：unset extraDirs 后 Hook 消失
   - 解决：重新配置 `hooks.internal.load.extraDirs`

### 最终配置

```json
{
  "hooks": {
    "internal": {
      "enabled": true,
      "load": {
        "extraDirs": ["/workspace/projects/hooks-pack"]
      }
    }
  }
}
```

### 关键命令

```bash
# 检查 Hook 状态
openclaw hooks list

# 安装 Hook Pack
openclaw hooks install -l /path/to/hooks-pack

# 重启 Gateway
sh /workspace/projects/scripts/restart.sh

# 查看日志
tail -f /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log
```

### 容器环境注意事项

- systemd 不可用是正常现象
- 使用 `sh scripts/restart.sh` 而非 `openclaw gateway restart`
- 日志中的 systemd 错误可忽略
