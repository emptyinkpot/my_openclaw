## 二、事件流程闭环

### 代码写入流程

```
AI 决定写入代码
        │
        ▼
┌───────────────────────────────────────┐
│ tool:call 事件触发                      │
│                                        │
│  ① pre-write-check (优先级 1)          │
│     → 提醒检查知识库（不阻止）           │
│                                        │
│  ② storage-protect (优先级 5)          │
│     → 检查是否删除受保护文件             │
│     → 如是，阻止并提示                  │
│                                        │
│  ③ code-guard (优先级 5)               │
│     → 检查代码是否违规                  │
│     → 如严重违规，阻止并提示             │
│     → 如警告，允许但提示                │
└───────────────────────────────────────┘
        │
        ▼
代码写入（如未被阻止）
        │
        ▼
┌───────────────────────────────────────┐
│ tool:complete 事件触发                  │
│                                        │
│  ④ task-logger (优先级 200)            │
│     → 记录操作到日志                    │
│     → 执行预设任务                      │
└───────────────────────────────────────┘
```

### 消息处理流程

```
用户发送消息
        │
        ▼
┌───────────────────────────────────────┐
│ message:received 事件触发               │
│                                        │
│  ① sender-trigger (优先级 50)          │
│     → 检查发送者是否在规则中             │
│     → 如是，执行预设任务                │
│                                        │
│  ② message-received-check-state        │
│     → 检测消息类型和意图                │
│     → 记录到日志                        │
└───────────────────────────────────────┘
        │
        ▼
AI 处理消息
```

### 任务完成/错误流程

```
任务完成 或 发生错误
        │
        ▼
┌───────────────────────────────────────┐
│ task:complete / error 事件触发          │
│                                        │
│  self-heal                             │
│    → 记录成功/失败经验                  │
│    → 错误时搜索知识库                   │
│    → 提供解决方案建议                   │
└───────────────────────────────────────┘
```

## 三、层级防护

### 第一层：Workspace Context（对话前）

- AGENTS.md → 定义每次会话必须读取的文件
- AUTO_SCRIPTS.md → 定义修改代码前的强制流程
- SOUL.md → 定义 AI 核心行为准则
- USER.md → 定义用户偏好和项目规范

**作用**：AI 在生成任何响应前，已经知道规范和要求

### 第二层：Hook 拦截（工具调用时）

- pre-write-check → 提醒检查知识库
- code-guard → 拦截严重违规代码
- storage-protect → 保护重要数据不被误删

**作用**：在 AI 执行危险操作前拦截或提醒

### 第三层：事后记录（工具调用后）

- task-logger → 记录所有操作
- self-heal → 记录经验，自动学习

**作用**：保留历史记录，便于复盘和学习

## 四、限制与约束

### ⚠️ 无法系统强制的场景

| 场景 | 原因 | 解决方案 |
|------|------|----------|
| AI 是否真正阅读知识库 | 发生在思考阶段，无法检测 | 通过 pre-write-check 提醒 |
| AI 是否遵守响应模板 | 无法验证 AI 输出格式 | 通过 SOUL.md 定义行为准则 |
| AI 是否正确理解规范 | 取决于模型能力 | 使用简洁明确的规范文字 |

### ✅ 可以系统强制的场景

| 场景 | 实现方式 |
|------|----------|
| 禁止 localStorage.clear() | code-guard 拦截 |
| 保护账号数据 | storage-protect 拦截 |
| 记录所有操作 | task-logger 记录 |

## 五、最佳实践建议

### 对于 AI

1. **每次对话开始**：自动读取 AGENTS.md 要求的文件
2. **修改代码前**：主动搜索知识库，检查已验证代码
3. **收到提醒后**：认真对待 Hook 的反馈和警告
4. **遇到错误时**：查看 self-heal 提供的历史经验

### 对于用户

1. **填写 IDENTITY.md**：定义 AI 的身份和性格
2. **更新 USER.md**：记录你的偏好和项目需求
3. **维护知识库**：定期更新 docs/ 目录下的文档
4. **配置 HEARTBEAT.md**：添加定期检查任务
