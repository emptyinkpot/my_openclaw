# 飞书查询小说状态 - 使用指南

## 功能概述

在飞书中直接查询小说更新、发布状态，无需登录服务器。

## 飞书命令

在飞书对话中 @机器人，发送以下命令：

### 1. 查询所有作品状态
```
@机器人 查询更新
@机器人 更新状态
@机器人 status
```

返回示例：
```
📚 小说更新状态
==================

📖 转生病弱王女的我被宿敌圈养了
   总章节: 80 | 已润色: 2
   审核通过: 0 | 待发布: 0
   今日操作: 2次 (polish)

📖 枪与凋零之花
   总章节: 99 | 已润色: 5
   审核通过: 1 | 待发布: 0
   今日操作: 5次 (polish)
```

### 2. 查询指定作品章节详情
```
@机器人 查询 枪与凋零之花
@机器人 查询 转生病弱王女
```

返回示例：
```
📖 枪与凋零之花 - 最近章节
==================

⏳ 95. 第九十五章
⏳ 96. 第九十六章
📝 97. 第九十七章
📝 98. 第九十八章
✅ 99. 第九十九章

图例: ✅可发布 ✓审核通过 📝已润色 ⏳待处理
```

### 3. 查询今日操作记录
```
@机器人 今日操作
```

返回示例：
```
📋 今日操作记录
==================

✅ 10:30 转生病弱王女 第3章 润色
✅ 11:15 枪与凋零之花 第5章 润色
```

### 4. 查询可发布章节
```
@机器人 可发布章节
@机器人 pending
```

返回示例：
```
📤 可发布章节
==================

📖 枪与凋零之花
   第2章 第二章

共 1 章待发布
```

## 状态图例说明

| 图标 | 含义 |
|------|------|
| ✅ | 可发布（已润色+审核通过） |
| ✓ | 审核通过 |
| 📝 | 已润色 |
| ⏳ | 待处理（未润色） |

## 技术实现

- **Skill 位置**: `workspace/skills/novel-status/`
- **数据源**: `/workspace/projects/auto-scripts/task-planner/data/chapters.db`
- **数据库表**: `works`, `chapters`, `action_log`

## 故障排查

如果查询无响应：
1. 检查飞书渠道状态: `openclaw channels status --probe`
2. 检查 skill 状态: `openclaw skills list | grep novel-status`
3. 重启 Gateway: `sh /workspace/projects/scripts/restart.sh`
