# Node.js 服务端口被占用问题排查

## 问题描述
novel-manager 服务重启时经常出现端口被占用错误。根本原因是旧进程没有完全退出，导致新进程无法绑定端口。重复犯这个错误是因为每次都用 pkill 但没检查是否真的杀死，而且没意识到旧进程还在监听端口。

## 解决方案
1. 使用 lsof -i :3001 查找占用进程 PID
2. 使用 kill -9 <PID> 强制杀死进程
3. 确认端口释放后再启动服务
4. 更好的方案是使用 pm2 或 systemctl 管理进程，避免手动启停
