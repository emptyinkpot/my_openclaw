
# 本次任务经验总结

## 核心问题总结
本次任务主要遇到的问题是**HTML/TypeScript 文件中的转义字符问题**，以及**路径引用错误**。

---

## 具体问题与解决方案

### 1. TypeScript 文件中的 HTML 转义字符问题
**问题表现**：
- `extensions/core/audit/` 目录下的 TypeScript 文件中，`&lt;`、`&gt;`、`&amp;` 等 HTML 实体没有被正确解码
- 导致 `SyntaxError: 'return' outside of function` 等语法错误
- novel-manager 插件无法加载

**解决方案**：
- 使用 sed 批量替换：`s/&amp;/\&/g; s/&lt;/</g; s/&gt;/>/g;`
- 或者使用 Node.js 脚本读取 git 原始 blob，直接写入，不做任何转义修改

---

### 2. index.html 中的中文乱码问题
**问题表现**：
- index.html 中的中文注释变成了乱码（如 `M-eM-^HM-^GM-fM-^MM-`）
- 导致浏览器端 `ReferenceError: switchNovelDataSubTab is not defined`

**解决方案**：
- 直接从 git 历史提交中读取原始 blob（`git cat-file -p <blob-hash>`）
- 避免通过工具修改 index.html，防止引入额外的编码问题

---

### 3. 模块路径引用错误
**问题表现**：
- `extensions/plugins/novel-manager/index.ts` 第 1806 行使用了 `require('./core/config')`
- 正确的路径应该是 `require('../../core/config')`
- 导致 `Error: Cannot find module './core/config'`

**解决方案**：
- 仔细检查相对路径：从 `extensions/plugins/novel-manager/` 到 `extensions/core/` 需要向上两级
- 使用 `../../core/config` 替代 `./core/config`

---

## 关键经验教训

### 1. 优先查看服务日志
- 遇到问题第一时间查看 `/app/work/logs/bypass/dev.log`
- 日志会直接指出错误位置和原因

### 2. 谨慎修改文件
- 避免盲目使用 `edit_file` 或 `write_file` 工具，可能引入转义问题
- 优先使用 `git show` 或 `git cat-file` 从 git 读取原始内容
- 读取 git 原始内容后直接写入，不做任何转义修改

### 3. 路径引用要仔细
- 相对路径要从当前文件位置开始计算
- 使用 `..` 向上导航目录
- 写完后立即验证，避免 MODULE_NOT_FOUND 错误

### 4. 不要频繁回滚
- 回滚会让用户感到困惑和不满
- 尽量在当前文件基础上定位和修复问题
- 只有在完全无法定位问题时才考虑回滚，且要明确告知用户

### 5. 验证要彻底
- 修复后不要只看服务是否启动
- 要检查：
  - 日志中是否有错误
  - curl 能否正常访问
  - 插件是否成功加载
  - API 路由是否注册成功
