## 1. 任务背景与目标

你提出的目标是：在不破坏现有 OpenClaw 结构前提下，整合 `https://github.com/superset-sh/superset`，并让它可以在 OpenClaw 内可访问、可联调、可观测状态。

本轮落地采用「桥接插件（superset-bridge）」方案，而不是直接重构 OpenClaw 主应用。

目标拆分如下：

1. OpenClaw 导航出现 Superset 入口。
2. 访问 `/superset/` 可进入 Superset 承载页。
3. Superset 前端请求通过 OpenClaw 代理前缀转发（`/superset/_app/*`）。
4. 能通过 API 判断上游 Superset 是否可用（health 检查）。
5. 插件配置可在 `openclaw.json` 管理（目标地址、超时）。

---

## 2. 最终方案（架构级）

### 2.1 接入方式

- 不直接改 Superset 源码逻辑作为 OpenClaw 子模块运行入口。
- 在 OpenClaw 的 `extensions` 下通过 `superset-bridge` 提供：
  - 页面路由
  - 反向代理
  - 健康检查 API
  - 配置读取

### 2.2 路由拓扑

- 页面入口：`/superset`、`/superset/`
- 代理入口：`/superset/_app/*`
- 管理 API：`/api/superset-bridge/health`、`/api/superset-bridge/config`

### 2.3 请求链路

用户访问 `/superset/` 后链路如下：

1. 命中插件页面路由，返回承载页 HTML。
2. 页面 JS 调用 `/api/superset-bridge/health`。
3. 若健康：iframe 加载 `/superset/_app/`。
4. `/superset/_app/*` 被插件代理到 `targetBaseUrl`（默认 `http://127.0.0.1:3000`）。
5. 遇到上游 30x 时，`Location` 被重写回 `/superset/_app/...`，避免跳出 OpenClaw 前缀。

## 3. 代码改动总览（按文件）

## 3.1 `projects/extensions/superset-bridge/backend/routes/http.ts`

这是本次核心改造文件，当前包含以下能力：

### A. 常量与运行时配置

- `SUPERSET_BRIDGE_PAGE_ROUTES = ['/superset', '/superset/']`
- `SUPERSET_BRIDGE_PROXY_PREFIX = '/superset/_app'`
- `SUPERSET_BRIDGE_API_PREFIX = '/api/superset-bridge'`
- 默认配置：
  - `targetBaseUrl: 'http://127.0.0.1:3000'`
  - `requestTimeoutMs: 15000`

并提供：

- `configureSupersetBridgeRuntime(api)`：从插件配置读运行参数
- `disposeSupersetBridgeRuntime()`：插件卸载时恢复默认配置

### B. 页面输出与导航注入

- 读取 `frontend/pages/superset/index.html` 作为页面源
- 尝试读取共享导航 `projects/shared/nav-bar.html`
- 自动把 Superset 入口设置为当前激活态（`class="on"`）
- 页面输出时禁用缓存（`Cache-Control/Pragma/Expires`）

### C. 代理能力

- 根据请求路径拼出目标 URL
- 透传方法、路径、查询串与 headers
- 增补 `x-forwarded-host`、`x-forwarded-proto`
- 处理 upstream timeout 与网络错误
- 返回统一 JSON 错误：提示 Superset 服务不可用

### D. 重定向头重写（关键）

- 函数：`rewriteLocationHeader`
- 重写策略：
  - 相对路径 `/xxx` -> `/superset/_app/xxx`
  - 与 `targetBaseUrl` 同源的绝对地址 -> 改写为代理前缀
  - 非同源地址保持原样

### E. 健康检查 API

- `GET /api/superset-bridge/health`
- 主动请求上游 `/`，判定 `200~499` 为可达
- 返回：`success/status/module/targetBaseUrl/proxyPrefix/pageRoutes/upstream/startupHint`

### F. 配置查询 API

- `GET /api/superset-bridge/config`
- 返回当前运行时配置（`targetBaseUrl` 与 `requestTimeoutMs`）

### G. 对外处理函数

- `handleSupersetBridgePage`
- `handleSupersetBridgeProxy`
- `handleSupersetBridgeApi`

> 这三个函数由插件入口统一注册到 OpenClaw 的 HTTP 路由系统。

## 4. 已处理错误与修复说明

### 4.1 TypeScript 报错（已修复）

报错现象：

- `res.writeHead(status, headers)` 处出现 overload 不匹配
- 根因是 `proxyRes.headers.location` 类型可能是 `string[]`
- `OutgoingHttpHeaders.location` 仅接受 `string | undefined`

修复措施：

1. 引入 `OutgoingHttpHeaders`
2. 将 `rewriteLocationHeader` 入参改为 `string | undefined`
3. 仅当 `proxyRes.headers.location` 为字符串时才重写

修复后结果：

- 该 TS 类型错误已消失
- `writeHead` 参数类型与 Node HTTP 定义一致

## 10. 稳定启停增强（2026-03-31 夜间补充）

针对“点击快捷方式拉起网关并打开页面；退出浏览器后自动关闭网关”的诉求，已补充稳定启停链路：

1. 新增运行时管理脚本：`openclaw-superset-runtime.ps1`
   - 统一支持 `start/stop/status`
   - 启动时自动检测 Bun、写入 PID、等待 3000/3001 Ready
   - 停止时按 PID + 端口双保险清理 Superset API/Web
2. 重构一键启动脚本：`start-openclaw.superset.bat`
   - 不再用 `start cmd /k` 方式散开多个终端，改为调用 runtime manager
   - 启动 Superset 成功后再调用 `start-openclaw.bat`
   - 若网关启动失败，自动回滚 Superset 进程，避免“半启动”
3. 增强浏览器退出自动停机：`openclaw-browser-watch.ps1`
   - 保留原有“关闭浏览器 -> 关闭网关”逻辑
   - 当环境变量 `OPENCLAW_SUPERSET_AUTO_STOP=1` 时，额外停止 Superset runtime
4. 新增一键关闭脚本：`stop-openclaw.superset.bat`
   - 主动执行 `openclaw gateway stop`
   - 之后执行 `openclaw-superset-runtime.ps1 -Action stop`

### 快捷方式建议

- 启动快捷方式目标：`e:\My Project\Atramenti-Console\start-openclaw.superset.bat`
- 关闭快捷方式目标：`e:\My Project\Atramenti-Console\stop-openclaw.superset.bat`

### 行为说明

- 点击启动快捷方式：拉起 Superset + OpenClaw，并自动打开页面。
- 直接关闭被脚本拉起的浏览器窗口：watcher 会自动关闭网关；并在 `OPENCLAW_SUPERSET_AUTO_STOP=1` 下同步关闭 Superset。
- 若你手动再开了其他 OpenClaw 受控窗口，watcher 会识别并避免误关。

## 11. 规范化与防回归约束（2026-03-31 深夜补充）

为确保“低耦合、高内聚、可持续维护”，本次增加了以下工程约束：

1. **职责边界固定**
   - `start-openclaw.superset.bat`：仅做编排（先 Superset 后 OpenClaw，失败回滚）。
   - `openclaw-superset-runtime.ps1`：仅做 Superset 进程生命周期管理（start/stop/status）。
   - `openclaw-browser-watch.ps1`：仅做“浏览器退出 -> 网关/可选 Superset 停止”。
   - 这样避免脚本之间功能重叠，降低连锁修改风险。

2. **并发互斥保护**
   - `openclaw-superset-runtime.ps1` 新增全局互斥锁 `Global\\OpenClawSupersetRuntimeManager`。
   - 防止 start/stop/status 并发执行时互相踩状态（PID 文件、端口探测、清理动作）。

3. **参数与环境稳定性约束**
   - 统一处理 `Root` 的引号与尾斜杠，避免 `Test-Path` 非法字符问题复发。
   - 常量化端口（3000/3001）并集中定义，避免散落硬编码。
   - 通过 `$ProgressPreference='SilentlyContinue'` 避免 PowerShell 进度输出污染日志和批处理解析。

4. **新增最小化自检脚本**
   - 新文件：`verify-openclaw-superset.ps1`
   - 校验项：
     - 启停脚本存在性
     - runtime 管理脚本存在性
     - 导航 `Superset` 入口存在性
     - 桌面快捷方式是否正确指向 `start-openclaw.superset.bat`
     - runtime `status` 命令是否可执行
   - 该脚本为“改动后必跑”的守门检查，降低后续改坏概率。

5. **后续改动红线（建议固化到 Code Review）**
   - 禁止在 `start-openclaw.superset.bat` 内直接新增复杂业务逻辑（只保留编排）。
   - 禁止在 watcher 中直接注入 Superset 启动逻辑（只允许 stop 协同）。
   - runtime 的 start/stop/status 输出字段保持兼容，避免外围脚本解析失效。
   - 每次改动后必须执行：`verify-openclaw-superset.ps1`。
