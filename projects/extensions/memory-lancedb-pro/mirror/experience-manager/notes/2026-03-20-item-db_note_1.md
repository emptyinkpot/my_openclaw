# 📚 项目技术栈与架构

## 1️⃣ 核心技术栈

| 层次 | 技术 | 作用 |
|------|------|------|
| 后端语言 | TypeScript | 类型安全的 JavaScript，提高代码可维护性 |
| 运行环境 | Node.js | 服务器端 JavaScript 运行时 |
| 数据库 | MySQL | 关系型数据库，存储小说、经验、笔记等数据 |
| 前端 | HTML + CSS + JavaScript | 构建用户界面 |
| 包管理 | pnpm | 快速、节省空间的 Node.js 包管理器 |
| 平台 | OpenClaw | 插件运行平台，提供 HTTP 服务器、路由等能力 |

## 2️⃣ 项目架构（以 novel-manager 插件为例）

```
extensions/novel-manager/
├── index.ts                    # 路由层：接收请求，分发路由
├── services/
│   └── novel-service.ts        # 服务层：业务逻辑
├── core/
│   ├── ContentPipeline.ts      # 核心逻辑层：内容处理
│   └── database.ts             # 数据库连接管理
└── public/
    ├── index.html              # 前端页面
    ├── experience.html
    └── ...
```

### 各层职责：

1. **路由层（index.ts）**
   - 注册 HTTP 路由
   - 拦截页面请求和 API 请求
   - 调用服务层处理业务逻辑
   - 返回响应给前端

2. **服务层（novel-service.ts）**
   - 实现业务逻辑（比如：添加笔记、获取经验列表）
   - 调用数据库层存取数据
   - 封装核心逻辑（比如：调用 ContentPipeline 发布小说）

3. **核心逻辑层（ContentPipeline.ts 等）**
   - 实现核心功能（比如：内容清洗、番茄小说发布）
   - 不依赖具体的路由或服务，可复用

4. **前端层（public/*.html）**
   - 用户界面
   - 调用后端 API
   - 展示数据和交互

## 🚀 有条理开发的方法

### 步骤 1：需求分析
- 用户需要什么功能？
- 数据结构怎么设计？

### 步骤 2：数据库设计
- 创建表结构
- 设计字段和索引

### 步骤 3：后端开发 - 服务层
- 实现业务逻辑方法
- 封装数据库操作

### 步骤 4：后端开发 - 路由层
- 添加 API 端点
- 处理请求和响应

### 步骤 5：前端开发
- 构建用户界面
- 调用后端 API
- 实现交互逻辑

### 步骤 6：测试和调试
- 重启服务
- 测试功能
- 检查数据

## 🎯 有条理开发的建议

1. **先想清楚再动手**
   - 需求是什么？
   - 数据结构怎么设计？
   - 接口怎么定义？

2. **分层次开发**
   - 先数据库，再后端，最后前端
   - 每一层做好自己的事，不要越界

3. **小步快跑**
   - 一个功能一个功能地做
   - 每做完一个就测试，确保没问题再继续

4. **保持代码规范**
   - 命名有意义
   - 函数不要太长，一个函数只做一件事
   - 写注释解释复杂的逻辑

5. **版本控制**
   - 使用 Git 管理代码
   - 每次完成一个功能就提交
   - 写清楚提交信息
