# File:[identity] 文件夹问题

用户询问：`File:[identity]` 这是原生文件夹吗？

---

## 回答

### ❌ 不是原生文件夹！

`File:[identity]` 不是真实的文件夹名称！

---

### 🔤 这是一个占位符

`File:[identity]` 看起来像是一个占位符或者模板变量！

这种格式通常表示：
- `File:` - 前缀，表示这是文件相关的
- `[identity]` - 方括号里的内容会被替换成实际的值

---

### 📝 可能出现的场景

这种占位符通常出现在：

#### 1. 配置文件示例中
```json
{
  "path": "File:[identity]/config.json"
}
```

#### 2. 文档中
```
配置文件位置：File:[identity]/settings.yaml
```

#### 3. 插件或技能的配置模板中
```
数据存储：File:[identity]/data/
```

---

### 🔍 举个例子

如果 `[identity]` 被替换成 `my-profile`，那么：

```
File:[identity]/config.json
```

就变成了：

```
File:my-profile/config.json
```

或者更常见的是直接变成：

```
my-profile/config.json
```
