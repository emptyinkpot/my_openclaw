# 经验模块云端主存化：本地 JSON 迁移到云端 MySQL

## Summary
经验模块从本地 JSON 迁移到云端 MySQL，运行时统一读取云端主存。

## Problem
如何让经验模块不再依赖本地数据，统一使用云端数据？

## Solution
引入 ExperienceCloudStore 作为单一存储层，启动时自动建表和补字段，Repository 改成异步 MySQL 读写，路由与 UI 状态同时兼容旧字段和云端主存状态。

## Applied
- 先看现成数据库层
- 让 source of truth 单一
- 迁移优先于推倒重建

## Gained
- 云端主存和外部同步要分层
- 旧表结构必须补字段而不是假设新表已生效
- 异步仓库需要同步改路由

## Verification
- GET /api/experience/stats 返回云端记录数
- GET /api/experience/cloud/status 显示 cloud primary
- 前端经验页可正常加载
