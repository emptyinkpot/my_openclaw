# 双网关问题：旧计划任务与桌面启动器同时起网

## Summary
双网关的本质是旧计划任务和桌面启动器并存；用无害 gateway.cmd stub 收敛成单入口。

## Problem
为什么会有两个网关，怎么把它收敛成一个？

## Solution
1. 确认唯一真实入口是桌面启动器与其 gateway-run 链路。
2. 找到旧的 OpenClaw Gateway 计划任务。
3. 因为当前用户权限无法直接禁用计划任务，就把 C:\Users\ASUS-KL\.openclaw\gateway.cmd 改成无害空壳，避免旧任务再起第二份网关。
4. 保留一个前台启动器、一个网关监听端口，避免双入口并发。
5. 把这次排障经验写入云端经验库，供后续复用。

## Applied
- 先查进程与计划任务，而不是只看页面表象
- 凡是双入口问题，先收敛入口再看功能
- 不能直接改系统权限时，用兼容壳阻断旧行为

## Gained
- 同一个服务如果同时存在旧任务和新启动器，最常见问题就是重复拉起与锁冲突
- 无权限环境下，兼容 stub 比硬删计划任务更稳
- 经验库应该记录“根因 + 收敛动作 + 验证方式”，而不是只记现象

## Verification
- schtasks /Query 显示旧任务仍存在
- gateway.cmd 现已改成 no-op stub
- 当前仅有一个 127.0.0.1:5000 监听进程
