## 使用指南

### 立即更新所有文档
```bash
cd /workspace/projects/docs/tools

# 更新脚本清单、变更日志、任务列表
node generate-docs.js

# 更新 API 文档
node auto-generate-api.js

# 检测变更并更新
node watch-and-sync.js
```

### 添加到 Git Hook（推荐）
```bash
# 在 .git/hooks/post-commit 中添加
node /workspace/projects/docs/tools/generate-docs.js
```

### 定时自动更新
```bash
# 添加到 crontab，每小时更新
0 * * * * cd /workspace/projects && node docs/tools/watch-and-sync.js
```

## 为什么不能 100% 自动化？

| 人工维护项 | 原因 |
|-----------|------|
| **Bug 记录** | 现象描述、根因分析、经验总结需要人类判断 |
| **技术决策** | 方案对比、权衡分析、上下文信息无法从代码提取 |
| **架构设计** | 抽象思维、模块关系、演进规划需要设计能力 |

> 💡 **经验法则**: 能从代码/日志直接提取的信息 → 全自动  
> 💡 **知识沉淀**: 需要理解、分析、总结的信息 → 人工维护
