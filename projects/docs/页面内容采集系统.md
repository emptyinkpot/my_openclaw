# 页面内容采集系统

## 问题背景

当 AI 需要操作浏览器时，传统方式是：
1. AI 思考需要什么元素
2. 编写选择器
3. 执行查找
4. 如果找不到，重新思考

这个过程效率低，而且 AI 无法"看到"页面内容。

## 解决方案

### 方案一：DOM 快照 + 元素枚举

**实现文件**: `auto-scripts/src/utils/content-extractor.js`

**功能**:
- `getDOMSnapshot()` - 获取完整 DOM 树
- `enumerateInteractiveElements()` - 枚举所有可交互元素
- `getPageSummary()` - 生成页面结构摘要
- `generateAIDescription()` - 生成 AI 友好的描述

**使用方式**:
```javascript
const { ContentExtractor } = require('./utils/content-extractor');

const extractor = new ContentExtractor(page);

// 获取页面摘要
const summary = await extractor.getPageSummary();
// { url, title, headings, regions, activeDialogs, focusedElement }

// 获取可交互元素
const elements = await extractor.enumerateInteractiveElements();
// { totalElements, elements: [{ type, selectors, text, placeholder, visible }] }

// 生成 AI 描述
const aiDesc = await extractor.generateAIDescription();
```

### 方案二：Accessibility Tree (推荐)

**实现文件**: `auto-scripts/src/utils/accessibility-tree.js`

**功能**:
- 使用 Chrome DevTools Protocol 获取无障碍树
- 无障碍树比 DOM 更结构化
- 自动过滤隐藏元素和装饰性元素
- 更适合 AI 理解

**使用方式**:
```javascript
const { PageAnalyzer, AccessibilityTreeCollector } = require('./utils/accessibility-tree');

// 快速分析
const analyzer = new PageAnalyzer(page);
const quick = await analyzer.quickAnalyze();
// { url, title, headings, regions, interactiveCount, interactiveElements }

// 生成 AI 提示
const prompt = await analyzer.generateAIPrompt();
```

### 方案三：CDP 实时监控

通过 Chrome DevTools Protocol 连接到运行中的浏览器，实时监控页面变化：

```javascript
// 连接到已有浏览器
const browser = await chromium.connectOverCDP('http://localhost:9222');
const page = browser.contexts()[0].pages()[0];

// 监控页面变化
page.on('framenavigated', async () => {
  const elements = await extractor.enumerateInteractiveElements();
  console.log('页面元素:', elements.totalElements);
});
```

## 技能集成

**路径**: `workspace/skills/analyze-page/`

**用法**:

```bash
node workspace/skills/analyze-page/scripts/analyze.js quick

node workspace/skills/analyze-page/scripts/analyze.js watch
```

**路径**: `workspace/skills/page-explorer/`

**用法**:

## 工作流程

```
1. 用户请求操作页面
       ↓
2. AI 调用 analyze-page 技能
       ↓
3. 获取页面结构和元素列表
       ↓
4. AI 分析并选择正确的选择器
       ↓
5. 执行操作
```

## 输出文件

所有快照保存在 `/workspace/projects/storage/page-snapshots/`:

- `snapshot-{timestamp}.json` - 完整快照
- `snapshot-{timestamp}-a11y.txt` - Accessibility Tree 描述

## 优势

1. **效率高**: 一次获取所有元素，无需逐个查找
2. **准确性**: 使用多种选择器策略，提高定位成功率
3. **AI 友好**: 结构化输出，便于 AI 理解
4. **实时性**: 支持监控页面变化
5. **可追溯**: 快照保存，便于调试
